## 5. Δομή του κώδικα

Σύντομη περιγραφή των αρχείων:

```text
efp1-2025-assignment-team15/    # Ρίζα του project
.
├── main.py         # Είσοδος του προγράμματος / μενού 
├── models.py       # Κλάσεις τομέα: Employee, Driver, Palette, ProductCategory, Notification
├── services.py     # Λογική συστήματος 
└── diagrams/
    ├── class_diagram.png
    └── sequence_diagram.png
```

Περιγράψτε με 1–2 προτάσεις τι κάνει το κάθε αρχείο:

 - `main.py`:
    - περιέχει το μενού,
    - διαβάζει input από τον χρήστη,
    - καλεί τις συναρτήσεις από services.py.

- `models.py`:
    - περιέχει τις βασικές κλάσεις του συστήματος (οντότητες).

- `services.py`:
    - περιέχει συναρτήσεις που υλοποιούν τα βήματα του use case (δημιουργία/αναζήτηση/ενημέρωση αντικειμένων).

## 6. UML Διαγράμματα

 - `class_diagram.png` – class diagram των βασικών κλάσεων που υλοποιούνται.

 - `sequence_diagram.png` – sequence diagram για το βασικό use case.

Μπορείτε να προσθέσετε και άλλα (π.χ. .puml, .pdf) αν θέλετε.

## 8. Γνωστά Περιοριστικά Σημεία / Μελλοντική Επέκταση

Προαιρετικό, αλλά καλό αν θέλετε να δείξετε ότι σκέφτεστε σαν developers:

- Τι ΔΕΝ υποστηρίζει ακόμα το σύστημα (π.χ. δεν αποθηκεύει σε αρχείο, δεν έχει έλεγχο για invalid ημερομηνίες).
- Τι θα προσθέτατε σε επόμενη έκδοση.

*Παράδειγμα*:

- Προς το παρόν τα δεδομένα χάνονται όταν κλείσει το πρόγραμμα (δεν υπάρχει αποθήκευση σε αρχείο).
- Δεν γίνεται έλεγχος αν η ημερομηνία λήξης είναι πριν από την ημερομηνία έναρξης.
- Σε επόμενη έκδοση θα μπορούσαμε να προσθέσουμε αποθήκευση σε JSON/CSV και απλό έλεγχο ημερομηνιών.
